三、本地控制怎么做

必须审批、审计、超时、权限边界

控制请求流

1

远端 agent 需要操作本机时，只能发起结构化请求，例如 control.request。

2

Local Server 校验请求来源、agentId、动作类型、参数和风险级别。

3

Web UI 弹出审批卡片：允许一次、拒绝、仅预览、限定目录/窗口。

4

审批通过后由本地 control service 执行，并把结果回传远端 agent。

本地控制能力分层

L1

只读：截图、窗口标题、文件列表、日志读取。

L2

低风险写入：保存文件到授权目录、复制文本到剪贴板。

L3

交互操作：点击、输入、打开应用、浏览器操作，需要明确审批。

L4

高风险操作：删除文件、安装程序、发送敏感信息，默认禁用或二次确认。

四、当前必须修正的连接策略

避免误连本机 Gateway

当前风险

本机 18789 已经被 moltbot gateway 占用，Local Server 以为 tunnel 已存在，实际连到了本机。

推荐设置

gatewayLocalPort = 18790，远端仍为 gatewayRemotePort = 18789。

SSH Tunnel

127.0.0.1:18790 -> 100.74.38.97:127.0.0.1:18789，只把远端 Gateway 映射到隔离端口。

诊断规则

health 里必须显示 tunnel 进程来源；如果本地端口已被非 tunnel 进程占用，应提示冲突。

五、最终目标状态

聊天

用户在本机 UI 选择远端 agent，与对应 session 流式聊天。

文件

本机上传附件到远端 workspace，远端生成文件后可安全下载。

全量 Agent

Gateway configured agents + sessions + SSH CLI discovery 合并展示。

本地控制

远端 agent 只能通过审批队列控制本机，所有动作可追踪、可拒绝、可超时。
