detaches_agent v0.1 功能说明

当前版本实现了本地 Web UI 连接远端 OpenClaw Gateway，并为每个聊天会话绑定一个本机持久 Terminal。远端 agent 可以通过约定命令块请求控制用户电脑，用户审批后才会执行。

版本：v0.1 first commit 远端 Gateway：OpenClaw 本机控制：per-session Terminal 执行策略：用户审批

核心能力

多 Agent 聊天

本地 UI 通过本地 Node 后端连接 OpenClaw Gateway，读取远端 agent 列表，用户可选择不同 agent 进行聊天。

会话隔离

默认使用 agent:<agentId>:detaches:<deviceId> 独立会话，避免污染远端 main 会话；UI 可手动切换主会话。

本机持久 Terminal

每个聊天 sessionKey 绑定一个本机 terminal。切换会话时复用同一个 terminal；WebSocket 断开后后端仍保留该 terminal 进程和输出回放。

默认隐藏，可展开查看

聊天框下方有 Agent Terminal 折叠条，默认隐藏；用户可随时展开查看该 agent 对应 terminal 的活动。

Agent 请求控制本机

后端会把 detaches_agent 接入上下文附加到用户消息，让远端 agent 知道当前对话拥有可审批的本机 terminal。

审批后执行

agent 不能直接执行命令，只能返回 detaches-terminal 命令块。UI 展示审批卡，用户点击 Run 后才写入对应 terminal。

当前链路

用户 Web UI
→
本地 Node Server
→
OpenClaw Gateway

聊天走 Gateway；本机执行走本地 Node Server 的 per-session Terminal WebSocket。

Agent 控制协议

远端 agent 如需控制用户电脑，必须返回如下 fenced code block：

```detaches-terminal
{"command":"pwd","reason":"查看用户本机当前工作目录"}
```
  • command：要写入本机会话 terminal 的命令。
  • reason：给用户看的执行原因。
  • UI 会把该块渲染成审批卡；用户点击 Run 后才执行。

运行方式

cd /Users/zhangshutong/code/detaches_agent
pnpm dev
  • 默认后端：http://127.0.0.1:38888
  • 默认前端：http://127.0.0.1:5173
  • 如果端口被占用，Vite 会自动切换到下一个可用端口，例如 5174。

已验证内容

  • pnpm typecheck 通过。
  • pnpm build 通过。
  • 同一 session 的 terminal 断开后重连，terminalId 不变，历史输出仍在 replay 中。
  • 本机 terminal 可接收 WebSocket 写入命令并返回输出。

当前限制与下一步

  • 当前环境中 node-pty 会 fallback 到 pipe shell；普通命令可用，全屏 TUI 体验后续可继续优化。
  • 建议下一版加入执行审计日志、命令风险提示、workspace 权限边界和一键停止任务。
  • 如果需要远端 terminal 模式，可在当前 per-session terminal 架构上增加目标类型：local / remote。
