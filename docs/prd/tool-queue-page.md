# Tool Queue Page

## Goal

Move Tool Queue out of the main chat sidebar into a dedicated page so approvals remain available without competing with the conversation layout.

## Product Behavior

- Add a top-level `工具队列` view next to `聊天` and `连接设置`.
- The page shows tool requests for the currently selected `sessionKey` and `agentId`.
- Keep existing request capabilities: approve/run, reject, view terminal/log, password entry for Main Agent file transfer, transfer progress, and result-forward retry.
- Pending requests may still surface an approval dialog even if the user is not already on the Tool Queue page.
- The main chat page no longer contains a persistent Tool Queue column.

## Request States

- `pending`: user can approve or reject.
- `approved` / running transfer: show progress or latest execution snapshot.
- `failed`: user can retry supported actions.
- `rejected` / completed historical requests remain visible only if the existing queue visibility rules include them.

## Known Follow-Up Areas

- Reduce duplicate transfer cards generated by repeated save-file requests.
- Improve result-forward reliability when a terminal or broker forward has not started.
- Add clearer grouping by request target and risk.
- Add aging/cleanup policy for stale requests.

## Requirements

- The Tool Queue page must not depend on the removed third column.
- `查看日志` should still reveal the chat terminal, even if that requires switching back to chat first.
- This change is a layout migration only; it does not rewrite the broker execution model.
