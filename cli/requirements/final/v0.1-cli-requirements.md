# Detach Agent CLI Requirements v0.1

## Purpose

Detach Agent CLI v0.1 provides a small command-line surface for using an already configured Detach Agent app connection.

The CLI must not become a second settings UI. It should reuse the running app and its active connection state whenever possible.

Detach Agent CLI is an app companion CLI. It requires the Detach Agent App local server.
It is not a standalone Gateway client and will not become one in future versions.

CLI 是 Detach Agent App 的伴生命令行入口，必须依赖 App 启动的本地 server。CLI 不会演进为独立客户端。

## Document Layout

CLI requirement documents are stored under `cli/requirements/`:

- `final/`: finalized requirements for a Detach Agent major/minor version.
- `archive/`: intermediate archived requirement documents worth keeping.
- `todo/`: pending CLI requirement notes and follow-up work.

Temporary drafts are not stored in this repository.

This document belongs to Detach Agent `v0.1.x`.

## Scope

v0.1 supports:

- Get the active agent connection status.
- List agents visible through the current Detach Agent app connection.
- Send a message to an agent session.
- Listen for agent processing result data.

v0.1 does not support:

- Starting or stopping the Detach Agent app/server.
- Standalone Gateway client mode.
- Direct Gateway connection from the CLI.
- Editing the full app configuration.
- Managing app profiles from the CLI.
- Tool queue approval.
- File upload or transfer.
- Adapter installation or distribution.

## Command Surface

Executable name:

```bash
detach-agent
```

Commands:

```bash
detach-agent --version
detach-agent help [command]

detach-agent agent status [--json]
detach-agent agent list [--json]
detach-agent agent send <agent-id-or-session-key> --message <text> [--local-control] [--wait] [--timeout-ms <ms>] [--json]
detach-agent agent listen <agent-id-or-session-key> [--run-id <runId>] [--timeout-ms <ms>] [--json] [--raw]
```

`config` is a reserved namespace for future CLI features. It must not be used in v0.1.

## Shared App Configuration

The CLI shares configuration with the Detach Agent app by talking to the local app server.

Default base URL:

```text
http://127.0.0.1:38888
```

Override order:

1. `--base-url <url>`
2. `DETACH_AGENT_BASE_URL`
3. Default base URL

`--base-url` and `DETACH_AGENT_BASE_URL` only override the Detach Agent App local server address. They must not be interpreted as Gateway addresses.

The CLI must not maintain its own agent config file in v0.1.

If the Detach Agent app is already connected to an agent, the CLI can directly send messages using that active connection.

If the local app server is unreachable, the CLI should fail with:

```text
Detach Agent app server is unreachable at http://127.0.0.1:38888. Open Detach Agent App first.
```

The exit code must be `3`. v0.1 and future versions must not try to start the app/server.

## Command Behavior

### `agent status`

Returns the current app-side connection status for the active configuration.

Expected data sources:

- `GET /api/health`
- `GET /api/gateway/status`

Human output should summarize:

- Local server state.
- Gateway connection state.
- Current transport summary.
- Agent terminal/local-control readiness when available.

`--json` should return the raw normalized status payload.

### `agent list`

Lists agents visible through the current app connection.

Expected data source:

- `GET /api/agents`

Human output should include:

- Agent id.
- Title.
- Status.
- Session key.
- Updated time when available.

This command lists real Gateway/OpenClaw agents. It is not related to Detach Agent app profiles.

### `agent send`

Sends a message to an agent session through the app server.

Input target:

- If `<agent-id-or-session-key>` matches an agent id from `agent list`, resolve it to that agent's `sessionKey`.
- Otherwise treat it as a direct `sessionKey`.

Expected transport:

- WebSocket `/api/chat/:sessionKey`

Client event:

```json
{
  "type": "send",
  "message": "<text>",
  "includeLocalControlContext": false
}
```

When `--local-control` is present:

```json
{
  "type": "send",
  "message": "<text>",
  "includeLocalControlContext": true,
  "activationReason": "user-click"
}
```

Success output should include:

- Target agent id when resolved.
- Session key.
- Gateway run id when available.

`--json` should return a stable machine-readable object with `ok`, `agentId`, `sessionKey`, `runId`, and `raw`.

When `--wait` is present, the command should keep listening on the same chat WebSocket for agent result events after the send acknowledgement.

Default wait timeout:

```text
120000
```

Users may override this with `--timeout-ms <ms>`. `--timeout-ms 0` means no automatic timeout and is allowed for complex long-running tasks.

### `agent listen`

Listens for agent result events from an existing agent session.

Input target:

- If `<agent-id-or-session-key>` matches an agent id from `agent list`, resolve it to that agent's `sessionKey`.
- Otherwise treat it as a direct `sessionKey`.

Expected transport:

- WebSocket `/api/chat/:sessionKey`

When `--run-id <runId>` is present, the CLI should register that run id with the socket before waiting:

```json
{
  "type": "track-run",
  "runId": "<runId>"
}
```

`--timeout-ms <ms>` controls the maximum listen time. `--timeout-ms 0` means no automatic timeout.

`--raw` prints raw WebSocket events for debugging or custom script processing.

## Output and Errors

Default output is human-readable.

`--json` is required for automation and must avoid decorative text.

Recommended exit codes:

- `0`: success.
- `1`: operation failed.
- `2`: invalid arguments.
- `3`: local Detach Agent app server unreachable.
- `4`: Gateway connection/auth/pairing issue.
- `5`: requested agent/session not found.

Sensitive fields must not be printed. CLI v0.1 does not expose auth token or password values.

## Acceptance Criteria

- The CLI command set contains only `agent status`, `agent list`, `agent send`, and `agent listen` plus help/version.
- The CLI is documented as an app companion CLI and not a standalone Gateway client.
- No v0.1 command edits the full app configuration.
- No v0.1 command exposes profile list/use operations.
- `agent list` is clearly documented as Gateway/OpenClaw agent listing, not app profile listing.
- `agent send` can send through an app connection that was already established in the Detach Agent UI.
- `agent send --wait` can wait for agent result events and supports user-defined timeout.
- `agent listen` can listen for session or run result events and supports user-defined timeout.
- The CLI has no separate config file in v0.1.
