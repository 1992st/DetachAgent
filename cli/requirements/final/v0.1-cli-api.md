# Detach Agent CLI API v0.1

Detach Agent CLI is an app companion CLI. It requires the Detach Agent App local server.
It is not a standalone Gateway client and will not become one in future versions.

CLI 是 Detach Agent App 的伴生命令行入口，必须依赖 App 启动的本地 server。CLI 不会演进为独立客户端。

## Global Options

```bash
--base-url <url>
--json
```

Default base URL:

```text
http://127.0.0.1:38888
```

Environment variable:

```bash
DETACH_AGENT_BASE_URL=http://127.0.0.1:38888
```

`--base-url` has priority over `DETACH_AGENT_BASE_URL`.

`--base-url` and `DETACH_AGENT_BASE_URL` only set the Detach Agent App local server address. They are not Gateway addresses.

## Commands

```bash
detach-agent --version
detach-agent help [command]

detach-agent agent status [--json]
detach-agent agent list [--json]
detach-agent agent send <agent-id-or-session-key> --message <text> [--local-control] [--wait] [--timeout-ms <ms>] [--json]
detach-agent agent listen <agent-id-or-session-key> [--run-id <runId>] [--timeout-ms <ms>] [--json] [--raw]
```

## `agent status`

```bash
detach-agent agent status
detach-agent agent status --json
```

Returns current Detach Agent app connection status.

## `agent list`

```bash
detach-agent agent list
detach-agent agent list --json
```

Returns agents visible through the current app connection.

## `agent send`

```bash
detach-agent agent send <agent-id-or-session-key> --message <text>
detach-agent agent send <agent-id-or-session-key> --message <text> --local-control
detach-agent agent send <agent-id-or-session-key> --message <text> --wait --timeout-ms <ms>
```

Options:

- `--message <text>`: message text to send.
- `--local-control`: include Detach Agent local-control context.
- `--wait`: wait for agent result events after sending.
- `--timeout-ms <ms>`: wait timeout. Default is `120000`. Use `0` for no automatic timeout.
- `--json`: print JSON output.

Examples:

```bash
detach-agent agent send agent-123 --message "hello"
detach-agent agent send agent-123 --message "run task" --wait --timeout-ms 600000
```

JSON output:

```json
{
  "ok": true,
  "agentId": "agent-123",
  "sessionKey": "agent:agent-123",
  "runId": "run_abc",
  "timedOut": false,
  "events": []
}
```

## `agent listen`

```bash
detach-agent agent listen <agent-id-or-session-key>
detach-agent agent listen <agent-id-or-session-key> --run-id <runId>
detach-agent agent listen <agent-id-or-session-key> --run-id <runId> --timeout-ms 0
```

Options:

- `--run-id <runId>`: listen for a specific run.
- `--timeout-ms <ms>`: listen timeout. Default is `120000`. Use `0` for no automatic timeout.
- `--json`: print collected events as JSON.
- `--raw`: include raw WebSocket events.

Example:

```bash
detach-agent agent listen agent-123 --run-id run_abc --timeout-ms 0
```

JSON output:

```json
{
  "ok": true,
  "agentId": "agent-123",
  "sessionKey": "agent:agent-123",
  "runId": "run_abc",
  "timedOut": false,
  "events": []
}
```

## Exit Codes

- `0`: success.
- `1`: operation failed or timed out.
- `2`: invalid arguments.
- `3`: local Detach Agent app server unreachable.
- `4`: Gateway connection/auth/pairing issue.
- `5`: requested agent/session not found.
