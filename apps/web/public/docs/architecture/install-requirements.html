Install Requirements

Detach Agent Skill 安装需求

安装是一个受控产品能力：识别 Agent 类型，选择正确 skill scope，发送带 zip 附件的安装请求，并用回执证明 Host/Main Agent 已能加载完整 skill。

安装结论

主流程只服务一件事：让 Host/Main Agent 可安装、索引、启用 detach-agent-relationship。安装流程必须能拿到 zip、知道安装 scope、判断版本，并返回可验证回执。

Target 安装到 Host 侧 skill scope

具体位置由 Agent 类型决定。OpenClaw 使用 shared managed/global scope；Codex 使用 $CODEX_HOME/skills。

Not Target 不安装到 Detach 机器

Detach Agent 是消息来源和执行环境，不是 relationship skill 的安装主体。

Package 安装完整 skill 包

zip 解压后必须是完整 detach-agent-relationship/ 目录，不是裸 SKILL.md。

Trigger 只在触发点出现

创建 Detach 窗口前、设置/诊断页、更新/修复时触发；普通 Main Agent 对话不触发。

安装与验证触发

创建 Detach Agent 窗口前先点“验证 Skill 安装”。返回 missing、outdated 或 broken 时，再点安装/更新按钮。这里生成的是发给 Host/Main Agent 的 payload。

安装 Skill 到 OpenClaw 全局路径 验证 Skill 安装

按钮跳转到对应 payload。接入真实 Agent API 后，由产品层把 payload 和 zip 附件发给 Host/Main Agent。

实现审查防线

这些是安装实现必须持续验证的点。任何一项回退，都会让 Main Agent 拿不到包、装错位置，或无法判断升级。

zip 结构必须完整

detach-agent-relationship.skill.zip 必须包含 SKILL.md、VERSION 和 README.md。缺任意文件按 broken 处理。

打包脚本不能复制旧包

skill 包升级后必须先重建根目录 zip，再同步辅助包。Windows 包内 zip hash 必须等于根目录 zip。

安装 prompt 不能空泛

Prompt 必须告诉 Main Agent 从 attachment 读取 zip、按 Agent scope 安装、检查完整结构并返回回执。

manifest 不能保留旧 scope

Manifest 必须携带版本和 Agent-specific install policy。OpenClaw 默认 shared managed/global，不是 workspace。

搜索依据

安装路径规则必须来自 Agent 文档或本机可验证事实。未知路径不能被产品猜测。

OpenClaw Skills docs

确认 OpenClaw 有 workspace、project agent、personal agent、shared managed skills。全局安装使用 shared managed scope，典型路径是 ~/.openclaw/skills。

Codex skill-installer

确认 Codex skills 安装到 $CODEX_HOME/skills/<skill-name>，默认 ~/.codex/skills。

Agent 安装位置矩阵

同一个 skill，对不同 Agent 的安装 scope 不同。按钮、payload 和回执都必须携带 Agent 类型和 scope。

Agent 类型 安装 scope 安装位置 规则
OpenClaw shared managed / global ~/.openclaw/skills 或 OpenClaw 返回的 shared managed path 使用 global install，安装一次供本机 OpenClaw local agents 共享；除非用户明确选择 workspace-only，不装到 <workspace>/skills。
Codex Codex skills root $CODEX_HOME/skills，默认 ~/.codex/skills 按 Codex skill-installer 规则安装；安装后提示重启或刷新以加载新 skill。
Unknown unknown 不猜路径 先要求 Agent 返回 skill root，或询问用户确认路径；确认前不自动安装。

触发与状态机

安装检查发生在 Detach 窗口创建前，不进入普通 Main Agent 对话，也不被每条 marker 消息重复触发。

1. Create Window

用户创建 Detach Agent 窗口。系统先检查 Host 侧 skill 状态。

2. Resolve Agent

识别 Agent 类型、安装 scope、当前版本和生效路径。

3. Install / Update

missing、outdated、broken 时展示对应按钮。

4. Ready

状态为 ready 后发送 window bootstrap，后续消息只带 [[DETACH_AGENT]]。

missing

未安装，显示安装按钮。

outdated

版本低于目标版本，显示更新按钮。

broken

结构不完整或无法索引，显示修复按钮。

ready

已安装且可用，不展示安装流程。

安装/更新按钮

按钮不是固定文案，而是由 Agent 类型和状态共同决定。按钮旁必须显示 scope、版本和路径，避免用户误装。

安装到 OpenClaw 全局 Skills

OpenClaw + missing。目标是 shared managed/global scope。

更新 OpenClaw 全局 Skill

OpenClaw + outdated。覆盖同名 global skill 并刷新索引。

确认 Skill 安装位置

Unknown Agent。路径未确认前不执行安装。

验证 Skill 安装

安装后或诊断页触发。检查 skill 可见性、版本、完整结构、生效路径和刷新状态。

修复 Skill

broken 状态触发。重新发送完整 zip 并覆盖同名 skill。

重新检查

failed 或用户刷新状态时触发。不安装，只重新拉取 Host 侧状态。

状态 按钮行为 展示要求
checking 禁用按钮，显示检查中。 展示 Agent 类型和正在检查的 scope。
missing 启用安装。 展示目标版本、目标 scope、zip 名称。
outdated 启用更新。 展示当前版本和目标版本。
installing/updating 禁用按钮，显示进度。 展示 payload 已发送、等待回执。
failed 启用重试。 展示失败原因和下一步。

安装请求 Payload

Main Agent 不能靠 prompt 猜 zip。zip 必须作为同一安装请求的 attachment 传入；缺失或不可读时安装失败。

Receipt required

安装后必须返回版本、路径、结构和刷新状态。

Main Agent Prompt 设计

Prompt 的视角必须站在 Main Agent：它要知道自己是谁、从哪里拿包、按什么 Agent scope 安装、怎么判断成功。不要让它猜路径或猜包。

OpenClaw Prompt

You are the Host/Main Agent receiving a Detach Agent skill install request.

Install or update the attached complete skill package:
`detach-agent-relationship.skill.zip`

Rules:
1. Read the zip from this request's attachment. Do not guess or search for another zip.
2. For OpenClaw, install into the shared managed/global skills scope, normally `~/.openclaw/skills`, not a single workspace unless the user explicitly chooses workspace-only.
3. Do not install into the Detach Agent machine.
4. The extracted folder must be `detach-agent-relationship/` and include `SKILL.md`, `VERSION`, and `README.md`.
5. After install/update, refresh or reindex skills as required by OpenClaw.

If the target path is unclear, ask the user.

Return: skillName, oldSkillVersion, newSkillVersion, installedPath, installScope, packageStructureStatus, reloadOrReindexStatus.

Codex Prompt

You are the Host/Main Agent receiving a Detach Agent skill install request.

Install or update the attached complete skill package:
`detach-agent-relationship.skill.zip`

Rules:
1. Read the zip from this request's attachment. Do not guess or search for another zip.
2. For Codex, install into `$CODEX_HOME/skills`; if `CODEX_HOME` is unset, use `~/.codex/skills`.
3. Do not install into the Detach Agent machine.
4. The extracted folder must be `detach-agent-relationship/` and include `SKILL.md`, `VERSION`, and `README.md`.
5. Restart, refresh, or reindex Codex skills as required.

If the target path is unclear, ask the user.

Return: skillName, oldSkillVersion, newSkillVersion, installedPath, installScope, packageStructureStatus, reloadOrReindexStatus.

完整 Skill 包

安装对象是完整 skill，不是单独的 SKILL.md。运行时 prompt、bootstrap、marker 和安装器不进入 skill 包。

detach-agent-relationship/
  SKILL.md
  VERSION
  README.md
文件 职责
SKILL.md Main Agent 运行时规则。
VERSION skill 版本，用于安装和更新判断。
README.md 维护说明和变更边界。
不放 bootstrap

window bootstrap 是运行时启动消息，不属于 skill 包。

不放 marker

[[DETACH_AGENT]] marker 是消息协议资产，不属于 skill 包。

不放 installer

Windows installer、provider registry、安装 prompt 不属于 skill 包。

不放路线图

产品规划留在架构文档，不污染 Main Agent 运行时 skill。

打包与获取链路

Main Agent 获取 skill 包的路径必须由产品链路保证，不由模型猜测。HTML、manifest、根目录 zip、Windows 辅助包必须指向同一个版本。

1. Source Skill

skills/detach-agent-relationship/ 是唯一源目录，包含 SKILL.md、VERSION、README.md。

2. Build Zip

重建 detach-agent-relationship.skill.zip，zip 顶层必须是 detach-agent-relationship/。

3. Attach Payload

安装按钮发送 payload，并把该 zip 作为 attachment 同时传给 Main Agent。

4. Sync Aux Packages

Windows 包只作为辅助，必须复制同一个根目录 zip，不能单独维护第二份 skill。

Artifact 来源 验证
detach-agent-relationship.skill.zip skills/detach-agent-relationship/ zip 内含完整目录和版本文件。
detach-agent-skill-manifest.json 安装元数据 archiveName、skillVersion、packageVersion 与 zip 一致。
detach-agent-skill-installer.js 浏览器侧 payload 构造 payload attachment path 指向根目录 zip，并携带 Agent scope。
windows-installer/detach-agent-relationship.skill.zip 根目录 zip 复制 hash 必须与根目录 zip 一致。

更新管理

版本必须分层管理。只改文档不触发升级；protocol 要求更高 skillVersion 时才触发更新。

版本 来源 触发更新
skillVersion skill 包 VERSION 缺失、结构损坏、版本低于目标版本。
protocolVersion bootstrap / marker 语义 协议要求更高 skillVersion。
packageVersion zip 分发包 分发包内容升级且影响安装资产。
触发更新

缺失、旧版、结构不完整、protocol 需要更高版本、用户点击更新/修复。

不触发更新

普通 Main Agent 对话、后续 marker 消息、只改产品说明文档。

更新后回执

oldSkillVersion、newSkillVersion、installedPath、installScope。

刷新要求

安装后必须 refresh / reindex / 新 session，具体动作按 Agent 文档执行。

升级判定 动作 Main Agent 回执
未安装 安装目标 zip。 oldSkillVersion=null，返回新版本和路径。
已安装版本低于目标版本 覆盖同名 skill，保留目标 scope。 返回 old/new 版本、覆盖路径、刷新状态。
已安装版本相同但结构损坏 修复安装，重新解压完整包。 返回 packageStructureStatus=repaired。
workspace skill 覆盖 global skill 提示当前生效路径，由用户选择移除覆盖或 workspace-only。 返回 effectivePath 和 conflictPath。

对抗性审查

这些不是边缘情况，是安装设计必须主动防住的失败路径。

错装到 Detach Agent

阻止。安装主体只能是 Host/Main Agent 所属 skill scope。

OpenClaw 错装 workspace

除非用户明确选择 workspace-only，否则使用 shared managed/global scope。

未带 zip 附件

失败，不让 Main Agent 猜路径或猜包来源。

Unknown Agent 路径未知

不猜。要求 Agent 返回 skill root 或询问用户。

session 未刷新

提示新 session、refresh 或 reindex。

allowlist 禁用

提示检查 Agent allowlist 或 skill 可见性策略。

workspace 覆盖 global

提示当前生效路径，因为 OpenClaw workspace skills 可能优先于 managed global。

长 prompt 污染普通对话

安装流程只在触发点出现，不进入普通 Main Agent 对话。

设计文档与代码分叉

验收必须检查 manifest、JS payload、prompt、zip 内容和 Windows 辅助包是否与本需求一致。

版本文件缺失

没有 VERSION 时不能判断升级，必须按 broken 处理。

空泛 prompt 误导 Main Agent

Prompt 必须包含 attachment、scope、结构、刷新和回执要求。

同名包多份分叉

根目录 zip 是唯一分发源；辅助包只能复制，不能手工维护。

验收标准

验收看用户是否能做对，而不是看页面是否写了很多说明。

路径正确

OpenClaw 默认 shared managed/global；Codex 默认 ~/.codex/skills；Unknown 不猜。

按钮正确

按钮按状态和 Agent 类型变化，且显示 scope、版本、路径。

附件正确

Main Agent 从同一 payload 的 attachment 获取 zip。

包正确

安装完整 skill 包，包含 SKILL.md、VERSION、README.md。

触发正确

创建 Detach 窗口前检查；普通对话和后续 marker 不触发。

回执正确

回执包含版本、路径、scope、结构状态和刷新状态。

风险可见

错装、路径未知、附件缺失、allowlist、覆盖优先级都有明确处理。

实现可推进

后续安装实现按该文档做，不再把 Windows installer 或 provider registry 当主流程。
